# 진행자 가이드

> 워크샵 진행자를 위한 내부 가이드입니다. 참가자와 공유하지 마십시오.

---

## 개요

이것은 agy-cli (Antigravity CLI)를 위한 **4개 모듈, 약 3.5시간 분량의 실습 워크샵**입니다. agy-cli를 평가하거나 도입하려는 엔지니어, 기술 리드, 솔루션 아키텍트 등 개발자 대상을 위해 설계되었습니다.

---

## 진행 형식

| 형식 | 모듈 | 소요 시간 |
| :-- | :-- | :-- |
| ⚡ 라이트닝 | 모듈 1 + 모듈 2 주요 내용 | 1.5시간 |
| 📋 표준 | 모듈 1 + 2 + 3 | 2.5시간 |
| 📦 전체 | 4개 모듈 전체 | 3.5시간 |
| 🏗️ 확장 | 모든 모듈 + 오픈 랩 | 5시간 |

---

## 워크숍 사전 체크리스트

- [ ] 참가자가 agy-cli를 설치하고 인증을 완료함 *([setup.md](setup.md) 참조)*
- [ ] 인증 세부 정보가 배포됨 (세션별로 다름 — agy-cli 팀에 확인)
- [ ] 참가자가 Git 및 적절한 데모 코드베이스를 보유함
- [ ] 진행자가 현재 agy-cli 버전에서 모든 실습을 처음부터 끝까지 실행해 봄
- [ ] 화면 공유 / 프로젝션 테스트 완료

!!! warning "인증은 가장 주요한 실패 원인입니다"
    세션 시작 30분 전에 항상 워크숍 사전 인증 확인을 실행하세요. 다음을 실행합니다:
    ```bash
    agy --print "Say READY" --print-timeout 30s
    ```
    참가자가 응답을 받지 못하는 경우, 시작하기 전에 중지하고 디버깅하세요.

## 모듈별 진행 노트

### 모듈 1 — SDLC 생산성 향상 (50분)

**핵심 메시지:** agy는 익숙하지 않은 코드베이스를 탐색할 때 발생하는 정신적 부담을 덜어줍니다. 단순한 자동 완성이 아닙니다. 무엇이든 물어볼 수 있는 수석 엔지니어입니다.

- **데모를 먼저, 실습은 나중에 진행하세요.** 참가자들에게 각자의 코드베이스에서 실습하도록 요청하기 전에, 본인의 코드베이스에서 섹션 1.1(코드 이해)을 라이브 데모로 보여주세요.
- **흔한 마찰:** 참가자들은 완벽한 프롬프트를 작성하려고 시도합니다. 자연어를 사용하도록 권장하세요. "이 코드베이스의 인증 아키텍처를 설명해 주세요"보다 "인증이 어떻게 작동하는지 알려줘"가 더 좋습니다.
- **AGENTS.md 모멘트:** 섹션 1.5는 가치가 높은 데모입니다. 화면에서 실시간으로 AGENTS.md를 생성하고, 다음 세션이 즉시 얼마나 더 똑똑해지는지 보여주세요.

### 모듈 2 — 플러그인 생태계 (45분)

**핵심 메시지:** 단일 명령어인 `agy plugin import gemini`를 사용하면 Gemini CLI에서 구축한 모든 것을 가져올 수 있습니다. 확장 프로그램에 투자한 내용이 그대로 유지됩니다.

- **`agy plugin import gemini` 라이브 데모** — 청중이 실제 출력을 볼 수 있게 하세요. 시각적으로 매우 매력적입니다.
- **검증 실습:** 플러그인 검증 섹션은 `samples/plugins/workshop-helpers/`에 있는 샘플 플러그인과 함께 사용할 때 가장 잘 작동합니다.
- **마켓플레이스 플레이스홀더:** `plugin install`에 대한 질문을 받으면, 곧 출시될 예정이라고 답하고 현재 방법인 `plugin import`로 안내하세요.

### 모듈 3 — DevOps 및 자동화 (40분)

**핵심 메시지:** `--print`는 탈출구입니다. agy 출력을 파이프(pipe)할 수 있게 되면, 무엇이든 자동화할 수 있습니다.

- **최고의 데모:** `git diff --cached | agy -p "Review for bugs."` — 모든 개발자가 즉시 그 가치를 알아봅니다.
- **CI/CD 섹션:** 전체 워크플로우를 실시간으로 작성하지 마세요. GitHub Actions YAML을 보여주고 `--dangerously-skip-permissions` 패턴을 설명하세요.
- 청중이 보안/컴플라이언스에 중점을 두지 않는 한 **샌드박스 심층 분석은 건너뛰세요**.

### 모듈 4 — 다중 에이전트 및 고급 (45분)

**핵심 메시지:** 서브에이전트와 /btw의 조합은 질적인 도약입니다. 이 부분에서 agy는 단순한 챗봇이 아닌 오케스트레이터가 됩니다.

- **서브에이전트 데모는 감탄을 자아내는 순간입니다.** 실시간으로 두 개의 에이전트를 생성하고, 두 개가 동시에 실행되는 모습을 보여주세요.
- **/btw 데모:** 다소 긴 작업(파일 리팩토링)을 시작한 다음, 작업 중간에 /btw를 사용하세요. 주입된 메모가 반영되는 동안 커서가 계속 움직이는 것을 참가자들에게 보여주세요.
- **스케줄링:** 패턴을 개념적으로 설명하고, 라이브 데모는 하지 마세요(워크숍에서는 지연 시간 때문에 어색해질 수 있습니다).

## 참가자 자주 묻는 질문

| 질문 | 답변 |
| :-- | :-- |
| "agy는 어떤 모델을 사용하나요?" | `/model`을 사용하여 확인하고 전환하세요. [모델 문서](https://www.antigravity.google/docs/models)를 참조하세요. |
| "Gemini CLI와 어떻게 다른가요?" | agy는 Gemini CLI와 Claude의 플러그인을 모두 연결하고, 네이티브 서브에이전트 오케스트레이션을 지원하며, `/btw`를 통한 작업 중 모델 스티어링을 제공합니다. 완전히 다른 제품입니다. |
| "내 API 키를 사용할 수 있나요?" | agy는 브라우저 기반의 Google 로그인을 사용합니다. 엔터프라이즈 사용자는 GCP 프로젝트를 연결합니다. [엔터프라이즈 문서](https://www.antigravity.google/docs/enterprise)를 참조하세요. |
| "코드가 Google로 전송되나요?" | 데이터 처리 세부 정보는 [FAQ](https://www.antigravity.google/docs/faq)를 참조하세요. |
| "훅은 어떻게 되나요?" | agy-cli는 `hooks.json`을 통해 훅을 지원합니다. [훅 문서](https://www.antigravity.google/docs/hooks)를 참조하세요. |
| "대화 로그는 어디에 저장되나요?" | `~/.gemini/antigravity-cli/conversations/` |

## 워크숍 중 문제 해결

| 증상 | 해결 방법 |
| :-- | :-- |
| `agy: command not found` | PATH를 확인하세요. `which agy` 또는 `which agy-cli`를 실행하세요. |
| 인증 오류 / 401 | 세션 자격 증명이 만료되었을 수 있습니다. 인증을 다시 배포하세요. |
| `agy plugin list` 오류 | `~/.gemini/antigravity-cli/`가 존재하는지 확인하세요. |
| 느린 응답 | 네트워크를 확인하세요. 유휴 상태 이후 첫 실행은 작업 공간 인덱싱으로 인해 더 느릴 수 있습니다. |
| 서브에이전트가 생성되지 않음 | 참가자가 대화형 모드인지 확인하세요(`--print` 아님). |

## 워크숍 이후

1. 표준 워크숍 피드백 양식을 사용하여 피드백을 수집합니다.
2. 관찰된 agy-cli 버그나 예기치 않은 동작을 기록합니다 — agy-cli 팀에 보고합니다.
3. 우회 방법이 필요했던 모든 실습은 `CONTRIBUTING.md`에 문서 업데이트 대상으로 표시해야 합니다.
