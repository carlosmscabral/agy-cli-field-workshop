# 진행자 가이드

> 워크숍 진행자를 위한 내부 가이드입니다. 참가자와 공유하지 마십시오.

---

## 개요

이것은 Antigravity CLI를 위한 **4개 모듈, 약 5.5시간 분량의 실습 워크숍**입니다. 이 워크숍은 Antigravity CLI를 평가하거나 도입하려는 엔지니어, 기술 리드, 솔루션 아키텍트 등 개발자 대상자를 위해 설계되었습니다.

!!! warning "Gemini CLI 종료: 2026년 6월 18일"
    Gemini CLI는 **2026년 6월 18일**에 수명이 종료됩니다. 참가자가 마이그레이션에 대해 질문하면 `agy plugin import gemini`를 안내하십시오. 이것이 주요 마이그레이션 경로입니다. 모든 Gemini CLI 플러그인은 단일 명령으로 이전됩니다.

## 워크숍 사전 체크리스트

- [ ] 참가자가 Antigravity CLI를 설치하고 인증을 완료함 *([setup.md](setup.md) 참조)*
- [ ] 인증 세부 정보 배포 완료 (세션별 — agy-cli 팀과 확인)
- [ ] 참가자가 Git 및 적절한 데모 코드베이스를 보유함
- [ ] 진행자가 현재 agy-cli 버전에서 모든 실습을 처음부터 끝까지 실행해 봄
- [ ] 화면 공유 / 프로젝션 테스트 완료
- [ ] 모듈 2의 경우: 참가자가 `dotnet` 또는 `mvn`을 실행할 수 있는지 확인 (또는 제공된 컨테이너 사용)
- [ ] 모듈 3의 경우: `pip install google-adk` 및 `gcloud auth application-default login`이 작동하는지 확인

!!! warning "인증은 가장 주요한 실패 원인입니다"
    세션 시작 30분 전에 항상 워크숍 사전 인증 확인을 실행하세요:
    ```bash
    agy --print "Say READY" --print-timeout 30s
    ```
    참가자가 응답을 받지 못하는 경우, 시작하기 전에 중지하고 디버깅하세요.

## 모듈별 진행 노트

### 모듈 1 — SDLC 생산성 향상 (75분)

**핵심 메시지:** agy는 익숙하지 않은 코드베이스를 파악하는 데 드는 정신적 부담을 덜어줍니다. 단순한 자동 완성이 아니라, 무엇이든 물어볼 수 있는 시니어 엔지니어와 같습니다.

- **데모 먼저, 실습은 나중에.** 참가자들에게 각자의 코드베이스에서 실습하도록 요청하기 전에, 본인의 코드베이스에서 섹션 1.1(코드 이해)을 라이브 데모로 보여주세요.
- **흔한 마찰 요소:** 참가자들은 완벽한 프롬프트를 작성하려고 노력합니다. 자연스러운 언어를 사용하도록 권장하세요. "이 코드베이스의 인증 아키텍처를 설명해 주세요"보다 "인증이 어떻게 작동하는지 알려줘"가 더 좋습니다.
- **AGENTS.md 모먼트:** 섹션 1.5는 가치가 높은 데모입니다. 화면에서 실시간으로 AGENTS.md를 생성하고 다음 세션이 즉시 얼마나 더 똑똑해지는지 보여주세요.
- **플러그인 가져오기 데모 (섹션 1.7):** `agy plugin import gemini`를 라이브로 실행하세요. 시각적인 결과물이 매우 인상적입니다. 참고: 가져오기 중에 **사용자 지정 테마는 조용히 삭제되며** 마이그레이션할 수 없습니다. 참가자가 테마가 왜 이전되지 않았는지 묻는다면, 이는 예상된 동작입니다. 오류가 발생한 것이 아니라 해당 구성 요소가 단순히 건너뛰어진 것입니다.

### 모듈 2 — 레거시 코드베이스 현대화 (90분)

**핵심 메시지:** 엄격한 모드(strict mode) + 셀프 온보딩은 일주일이 걸리는 마이그레이션을 체계적인 오후 작업으로 바꿔줍니다. 에이전트가 자체 컨텍스트를 작성한 다음 이를 실행합니다.

**라이브 데모 스크립트 (권장):**

1. .NET 또는 Java 대상 저장소 클론 (시간 절약을 위해 미리 완료해 둠)
2. 엄격한 모드 진입: `/permissions strict`
3. 조사 프롬프트 실행 — 에이전트가 전체 코드베이스를 읽는 모습을 참가자들에게 보여줌
4. 에이전트가 AGENTS.md를 생성하도록 함 — 실제 컨텍스트를 캡처했음을 보여주기 위해 소리 내어 읽어줌
5. `ctrl+g` — 편집기에서 생성된 계획을 열고, 사람이 제어할 수 있음을 보여주기 위해 눈에 띄는 수정 하나를 수행함
6. `request-review`로 전환하고, 1단계(Phase 1)만 실행함
7. `/rewind` 보여주기 — 문제가 발생할 경우 해당 단계를 되돌림
8. 전체 데모: 약 15분 소요, 이후 참가자들이 직접 실습함

- **흔한 질문:** "전체 마이그레이션을 수행할 수 있나요?" — 네, 하지만 그 가치는 단순히 실행되는 것을 지켜보는 것이 아니라 검토하고 스티어링하는 데 있습니다. 참가자들이 계획을 수정하도록 권장하세요.
- **진행자 시간 조절 참고:** 0~1단계는 참가자당 약 20분이 소요됩니다. 참가자들이 2단계를 진행하는 동안 돌아다니며 살펴보세요.

### 모듈 3 — SDK로 AGY 에이전트 구축 (90분)

**핵심 메시지:** CLI는 개인용입니다. SDK 에이전트는 팀 전체가 호출할 수 있는 전문가 서비스입니다.

- **설정 확인:** 시작하기 전에 모든 사람이 `google-adk`를 설치하고 Vertex AI 인증이 작동하는지 확인하세요. 이것이 가장 흔한 장애물입니다.
- **`adk web .` 모먼트:** 참가자들이 브라우저 UI에서 첫 번째 에이전트를 실행하게 되면 분위기가 바뀝니다. 에이전트가 도구에 반응하는 것을 보게 되기 때문입니다.
- **모델 선택 표:** 생성을 위해서는 Flash-lite를, 오케스트레이션을 위해서는 Pro를 강조하세요. 비용 절감 의식은 타협이 아니라 기능입니다.
- **실습 11 (파이프라인):** `SequentialAgent` + `BaseAgent` 가드 패턴은 핵심 아키텍처 인사이트입니다. 시작하기 전에 5분 정도 시간을 내어 비용이 들지 않는 가드(zero-cost guards)에 대해 설명하세요.

### 모듈 4 — 멀티 에이전트 및 고급 패턴 (60분)

**핵심 메시지:** 서브에이전트 + `/btw`는 질적인 도약입니다. 이 지점에서 agy는 단순한 챗봇이 아니라 오케스트레이터가 됩니다.

- **서브에이전트 데모는 감탄을 자아내는 순간입니다.** 라이브로 두 개의 에이전트를 생성하고, 둘 다 동시에 실행되는 모습을 보여주세요.
- **/btw 데모:** 약간 긴 작업(파일 리팩터링)을 시작한 다음, 작업 중간에 `/btw`를 사용하세요. 주입된 메모가 통합되는 동안 커서가 계속 움직이는 것을 참가자들에게 보여주세요.
- **스케줄링:** 패턴을 개념적으로 설명하고, 라이브 데모는 하지 마세요 (지연 시간으로 인해 워크숍에서 어색해질 수 있습니다).

## 참가자 공통 질문

| 질문 | 답변 |
| :-- | :-- |
| "agy는 어떤 모델을 사용하나요?" | `/model`을 사용하여 확인하고 전환하세요. [모델 문서](https://www.antigravity.google/docs/models)를 참조하세요. |
| "Gemini CLI와 어떻게 다른가요?" | agy는 Gemini CLI 및 Claude의 플러그인을 연결하고, 네이티브 서브에이전트 오케스트레이션과 `/btw` 작업 중 스티어링을 지원합니다. Gemini CLI는 2026년 6월 18일에 지원이 종료(EOL)됩니다. |
| "내 API 키를 사용할 수 있나요?" | agy는 브라우저 기반 Google 로그인을 사용합니다. 엔터프라이즈 사용자는 GCP 프로젝트를 연결합니다. [엔터프라이즈 문서](https://www.antigravity.google/docs/enterprise)를 참조하세요. |
| "코드가 Google로 전송되나요?" | 데이터 처리 세부 정보는 [FAQ](https://www.antigravity.google/docs/faq)를 참조하세요. |
| "훅은 어떻게 되나요?" | agy-cli는 `hooks.json`을 통해 훅을 지원합니다. [훅 문서](https://www.antigravity.google/docs/hooks)를 참조하세요. |
| "대화 로그는 어디에 저장되나요?" | `~/.gemini/antigravity-cli/conversations/` |
| "내 Gemini CLI 테마가 가져오기 되지 않았습니다." | 정상적인 동작입니다. 사용자 지정 테마는 `agy plugin import gemini` 중에 조용히 삭제됩니다. 스킬, MCP 서버 및 에이전트는 정상적으로 이관됩니다. |
| "SDK 에이전트를 Cloud Run에 배포할 수 있나요?" | 네 — `adk deploy cloud_run`을 사용하세요. 모듈 3의 섹션 3.6을 참조하세요. |

## 워크숍 중 문제 해결

| 증상 | 해결 방법 |
| :-- | :-- |
| `agy: command not found` | PATH를 확인하세요. `which agy` 또는 `which agy-cli`를 실행하세요. |
| 인증 오류 / 401 | 세션 자격 증명이 만료되었을 수 있습니다. 인증을 다시 배포하세요. |
| `agy plugin list` 오류 | `~/.gemini/antigravity-cli/`가 존재하는지 확인하세요. |
| 느린 응답 | 네트워크를 확인하세요. 유휴 상태 이후 첫 실행은 작업 공간 인덱싱으로 인해 더 느릴 수 있습니다. |
| 서브에이전트가 생성되지 않음 | 참가자가 대화형 모드인지 확인하세요(`--print` 아님). |
| `google-adk` 가져오기 오류 (M3) | venv가 활성화되어 있는지 확인하세요: `source .venv/bin/activate` |
| Vertex AI 403 (M3) | `gcloud auth application-default login`을 실행하고 `GOOGLE_CLOUD_PROJECT`가 설정되어 있는지 확인하세요. |

## 워크숍 이후

1. 표준 워크숍 피드백 양식을 사용하여 피드백을 수집합니다.
2. 관찰된 agy-cli 버그나 예기치 않은 동작을 기록하고 agy-cli 팀에 보고합니다.
3. 우회 방법이 필요했던 모든 실습은 `CONTRIBUTING.md`에 문서 업데이트가 필요하다고 표시해야 합니다.
