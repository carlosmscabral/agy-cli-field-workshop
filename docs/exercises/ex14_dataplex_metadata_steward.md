# Exercise 14: Dataplex Metadata Curation (Optional)

> **Duration:** 20 min | **Module:** 3 — ADK Agents with agents-cli

---

## Objective

Configure the `agy` CLI to connect directly to Google Cloud’s **native, managed remote Model Context Protocol (MCP) servers** for BigQuery and Dataplex Knowledge Catalog. Use interactive natural language turns in Cloud Shell to discover raw tables, audit customer chat transcripts for PII, assess overall sentiment, and attach governance Aspect tags directly to the catalog entry without writing a single line of SDK code.

---

## Setup

For this exercise, we operate within the Google Cloud Shell environment. Ensure you have the `gcloud` project context set to your active GCP project.

```bash
# 1. Enable BigQuery, Dataplex, and Developer Knowledge APIs
gcloud services enable \
    bigquery.googleapis.com \
    dataplex.googleapis.com \
    aiplatform.googleapis.com \
    developerknowledge.googleapis.com

# 2. Enable the Developer Knowledge remote MCP server in your project
gcloud beta services mcp enable developerknowledge.googleapis.com

# 3. Create the raw landing dataset inside BigQuery
bq --location=us-central1 mk -d raw_data_lake

# 4. Create the customer feedback raw table and seed it with mock transcripts
bq query --use_legacy_sql=false '
  CREATE OR REPLACE TABLE raw_data_lake.customer_feedback_raw (
    entry_id STRING,
    created_at TIMESTAMP,
    cust_val FLOAT64,
    usr_fb_txt STRING
  );
  
  INSERT INTO raw_data_lake.customer_feedback_raw (entry_id, created_at, cust_val, usr_fb_txt)
  VALUES
    ("101", CURRENT_TIMESTAMP(), 850.50, "Loved the new UI! But support was slow. Contact me at carl@example.com."),
    ("102", CURRENT_TIMESTAMP(), 120.00, "Product broke after 2 days. Terrible service, do not recommend!"),
    ("103", CURRENT_TIMESTAMP(), 4500.00, "Excellent enterprise support, very happy with the transition.");
'

# 5. Prepare the Dataplex Aspect Type schema
cat << 'EOF' > aspect_schema.json
{
  "name": "ai_steward_metadata",
  "type": "record",
  "recordFields": [
    {
      "name": "sentiment_summary",
      "type": "string",
      "index": 1,
      "annotations": {
        "displayName": "Customer Sentiment",
        "description": "Customer Sentiment Summary"
      }
    },
    {
      "name": "contains_pii",
      "type": "bool",
      "index": 2,
      "annotations": {
        "displayName": "Contains PII Data",
        "description": "Whether the table contains unmasked PII data"
      }
    },
    {
      "name": "steward_notes",
      "type": "string",
      "index": 3,
      "annotations": {
        "displayName": "AI Steward Curation Notes",
        "description": "Descriptive curation notes generated by AI"
      }
    }
  ]
}
EOF

# 6. Provision the regional Aspect Type in Dataplex Catalog
gcloud dataplex aspect-types create ai-steward-metadata \
    --location=us-central1 \
    --display-name="AI Curation Steward Aspect" \
    --description="Metadata generated automatically by the Antigravity Autonomous Data Steward agent." \
    --metadata-template-file-name="aspect_schema.json"
```

## Part 1: Register Native GCP Remote MCP Servers (5 min)

Antigravity natively integrates with the open-source Model Context Protocol (MCP). Because Google Cloud serves native, remote MCP endpoints for BigQuery, Dataplex, and developer documentation knowledge, we can configure our workspace `.agents/mcp_config.json` to mount them automatically.

1. Create a `.agents/` folder in your workspace root if it does not already exist:

   ```bash
   mkdir -p .agents
   ```

2. Save the following remote server configuration as `.agents/mcp_config.json`:

   ```json
   {
     "mcpServers": {
       "bigquery-remote": {
         "type": "sse",
         "serverUrl": "https://bigquery.googleapis.com/mcp"
       },
       "knowledge-catalog-remote": {
         "type": "sse",
         "serverUrl": "https://dataplex.googleapis.com/mcp"
       },
       "developer-knowledge-remote": {
         "type": "sse",
         "serverUrl": "https://developerknowledge.googleapis.com/mcp"
       }
     }
   }
   ```

> [!TIP]
> Run `/mcp` inside the interactive `agy` shell to verify that all three remote MCP servers are successfully mounted and active. The `developer-knowledge-remote` server allows the agent to search real-time official Google Cloud and Antigravity documentation to ground its workflows dynamically.

## Part 2: Interactive Curation via `agy` CLI (10 min)

Launch `agy` inside your workspace:

```bash
agy
```

At the `agy` prompt, use natural language conversation to execute the entire Data Steward workflow. Because you are inside Cloud Shell, the remote MCP servers automatically inherit your authenticated IAM session.

1. **Ground on Dataplex curation best practices**:

   ```text
   > Search the developer documentation for Dataplex Aspect Type and MetadataTemplate specifications to understand field naming rules.
   ```

   *Expected behavior*: The agent calls `developer-knowledge-remote.search_documents`, gets official schema guidelines (including case restrictions and required fields), and confirms that Aspect IDs must be lowercase with no underscores (e.g. `ai-steward-metadata`).

2. **Discover untagged catalog entry**:

   ```text
   > Search Dataplex Catalog for assets in project "your-gcp-project" and region "us-central1".
   ```

   *Expected behavior*: The agent will ask for permission, run `knowledge-catalog-remote.search_entries`, and locate `raw_data_lake.customer_feedback_raw`.

3. **Inspect and audit table rows**:

   ```text
   > Inspect the schema of raw_data_lake.customer_feedback_raw and query the first 3 rows to audit for PII and sentiment.
   ```

   *Expected behavior*: The agent will run BQ MCP tools to retrieve schema and query rows. It should report back that column `usr_fb_txt` contains unmasked emails (PII) and that the sentiment is mixed.

4. **Tag catalog asset**:

   ```text
   > Tag the table with 'ai-steward-metadata' setting sentiment to "Mixed", contains_pii to True, and steward_notes to "Detected raw emails inside customer transcript log."
   ```

   *Expected behavior*: The agent calls `knowledge-catalog-remote.create_or_update_aspect` to apply the tag directly to the Dataplex entry.

## Part 3: Verification & Cleanup (5 min)

1. Verify that the Metadata Aspect tag was attached successfully via the Dataplex UI or using `gcloud`:

   ```bash
   # Confirm the table is tagged in Dataplex
   gcloud dataplex entries describe "projects/your-gcp-project/locations/us-central1/entryGroups/@bigquery/entries/raw_data_lake.customer_feedback_raw"
   ```

2. Clean up resources to prevent cost pollution:

   ```bash
   # Delete the Dataplex aspect type
   gcloud dataplex aspect-types delete ai-steward-metadata --location=us-central1 --quiet

   # Force-delete the BigQuery raw dataset
   bq rm -r -f -d raw_data_lake
   ```
